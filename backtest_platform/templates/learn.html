研究工作流

平台应该按研究闭环使用，而不是只看单次高收益。

Research Loop
1 字段选择未充分利用且有经济解释的字段
2 假设写清楚动量、反转、流动性或风险补偿逻辑
3 表达式用时序处理和截面排序形成信号
4 单因子看 IC、PnL、回撤和换手
5 筛选按分组、指标和任意排序指标做 maxcorr 贪心
6 组合用等权和 LGB 做样本外对照
最稳的路径是：先让单因子有解释，再用相关性控制多样性，最后用 2023 OOS 结果判断组合是否值得进入候选库。

单因子回测

用于验证一个经济假设，不用于证明最终可提交。

Single Alpha

适合做什么

快速判断字段和表达式是否有方向性、稳定性、可解释性。单因子要重点看 IC 和 PnL 形态，而不是只看某一次年化。

常用模板

rank(ret_20d) -rank(returns) rank(ts_mean(volume,5)/ts_mean(volume,20)) rank(vwap_gap)
单因子如果 IC 不高但和已有库低相关，仍然可能在组合里有价值；反过来，高 IC 但 maxcorr 很高，可能只是重复暴露。

Alpha历史筛选与分组

历史页是从单因子库进入组合研究的主入口。

History Lab

经济分组

单因子按主要经济含义分成动量、反转、波动率、流动性、微观结构、规模、基本面、情绪/分析师。明显同时混入多类信号的表达式标为混合/剔除，默认不进入组合筛选。

多条件筛选

筛选条件同时生效。例如 |IC| > 0.03、Sharpe > 1、MaxCorr < 0.7、分组为微观结构，只有全部满足的因子才会显示。

多选研究

左侧多选后可查看两两相关性矩阵，也可以直接做等权组合和 LightGBM OOS。相关性矩阵是判断“高 IC 重复暴露”与“低相关互补暴露”的第一道闸。

推荐流程：先按经济分组和指标筛出候选，再看相关性矩阵，最后用等权和 LGB 分别验证线性叠加与非线性增益。

等权组合回测

用来验证低相关因子的线性叠加价值。

Equal Weight

Alpha 历史页勾选多个单因子后，等权组合会先对每个因子做截面标准化，再做平均。它不学习复杂非线性，因此更容易解释，也更能看出低相关因子是否真的互补。

combined = mean(zscore(alpha_1), zscore(alpha_2), ..., zscore(alpha_n))
  • 适合比较同样数量下，“低相关低 IC”与“高相关高 IC”哪组组合更稳。
  • 如果等权组合的回撤明显下降，说明多样性在发挥作用。
  • 如果等权组合没有改善，说明候选因子可能只是同一类暴露的重复表达。

LightGBM 样本外回测

用于测试非线性组合，但必须严格防止泄漏和过拟合。

OOS ML

训练集

2020-2022，默认最多 300 个唯一表达式，采样受内存预算控制。

Purge

训练结束到 OOS 开始之间默认留 5 日间隔，降低 5 日标签重叠泄漏。

测试集

2023 只做预测和评价，不参与训练和参数学习。

LGB 会自动对输入表达式去重，并跳过无法解析、常数或全缺失特征。结果里会显示请求因子数、去重后数量、实际训练数量、跳过特征数和特征重要性。

LGB 年化很高不等于可以直接提交。必须同时看 2023 OOS、回撤、特征重要性是否集中、是否只靠少数同类因子，以及组合和已有候选的相关性。

低相关贪心去重

Alpha 历史页可按任意研究指标排序，再做 maxcorr 约束筛选。

MaxCorr

输入 maxcorr、保留个数 和排序指标后，系统先在当前筛选结果中保留明确经济分组的单因子，再按指定指标排序。可用排序指标包括 |IC|、IC、Sharpe、Fitness、年化收益、Turnover、最大回撤和 MaxCorr。

sort by selected_metric desc for alpha in sorted: keep if max_corr(alpha, kept) <= maxcorr stop when len(kept) == keep_count

贪心结果会直接变成当前页面筛选和多选状态，随后可以立刻查看相关性矩阵，或用等权和 LightGBM 对同一批因子做组合回测。

因子构建方法

优先寻找新的经济暴露，而不是重复改写同一个公式。

Alpha Design

分钟结构

从 VWAP 偏离、开收盘结构、成交集中度、日内冲击恢复等角度构建，通常和纯日频动量相关性较低。

流动性与成交

成交额、换手、异常量、Amihud 冲击可以表达交易拥挤、冲击成本和短期反转。

波动率形态

下行波动、波动率比值、偏度、K线影线适合表达风险补偿和行为反转。

组合变换

先做简单可解释表达式，再用 zscore、rank、ts_rank、group_neutralize 做稳健化，不要一开始就堆复杂函数。

指标解释

每个指标回答的问题不同，要组合判断。

Metrics
指标含义使用方式
IC / Rank IC因子值与未来收益的相关判断方向性和排序能力。
年化超额Top10% 多头相对全市场的年化收益观察收益强度，必须和回撤一起看。
最大回撤PnL 从高点到低点的最大跌幅过滤脆弱、阶段性失效因子。
Sharpe / Sortino收益稳定性Sortino 更关注下行波动。
Turnover持仓变化速度过高可能难交易，过低可能响应慢。
maxcorr与已选因子的最大收益相关控制重复暴露，构建多样化候选库。

过拟合控制

速度快不代表质量低，但必须有防线。

Robustness
  • 候选筛选可以用全样本快速发现信号，但最终 LGB 必须使用 2020-2022 训练、2023 OOS 观察。
  • 不要只提交同一批高度相关的 LGB；每个 LGB 应该由不同字段族或不同经济假设支撑。
  • 特征重要性如果集中在几个相似表达式上，要回到单因子层面检查是否重复。
  • 高年化必须同时配合低回撤、稳定 IC、低 maxcorr 和合理换手。

AI接口与排障

右下角助手用于平台内问答，不替代研究判断。

Assistant

AI 助手会读取平台源码、字段列表和操作符说明回答问题。DeepSeek API 不可用时，系统会自动使用本地兜底知识回答 LGB、等权、单因子、maxcorr 和指标解释，避免前端只显示 API 错误。

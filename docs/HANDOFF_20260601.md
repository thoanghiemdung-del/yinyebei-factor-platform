## 一、项目概述

YYB 是 A 股量化因子回测平台（Flask + SQLite + numpy），位于 `D:\yyb\backtest_platform\`。

**核心功能**:
- 单因子 IS/OOS 分拆回测（IS=2020-2022, OOS=2023）
- 四种组合策略：等权(EW)、ICIR加权、Ridge正则化、LightGBM
- **矩阵级嵌套套娃**：组合因子可被其他组合复用（缓存矩阵 242×5515 float32）
- Web 研究界面：因子浏览、筛选、贪心去重、组合回测、详情弹窗

**数据**: 970 个交易日，5515 只 A 股，89 个数据字段。标签为未来 5 日收益。

## 四、套娃（组合嵌套）机制

这是平台最核心的创新功能。

### 写缓存（每个组合完成时）
```
app.py _compute_superalpha_inline → _add_to_history → 返回 UUID
  → np.save(f'cache/ew_{uuid}.npy', combined_matrix)
```
矩阵格式: (242, 5515) float32, ~5MB

### 读缓存（套娃时）
```
app.py Format B → 遇到 superalpha 表达式
  → 检查 cache/ew_{aid}.npy 是否存在
  → 存在: 添加到 cached_expr_map[str(idx)] = cache_path
  → 表达式列表中用 '__cached__' 占位
_compute_superalpha_inline → 遇到 '__cached__'
  → np.load(cache_path) → 直接作为因子矩阵
  → zscore → 加权求和 → 市值中性化 → 评估
```

### 与单因子的等价性
缓存矩阵就是 [242×5515] float32，和单因子的 OOS 矩阵完全一样。套娃时把它当普通因子处理，不区分来源。

### 已知限制
- 嵌套层数不限，但高层嵌套的 Sharpe 提升边际递减（顶级组合间相关性 0.7-0.8）
- 跨风格嵌套（微结构+动量）相关性低（~0.03），IC 更高但 Sharpe 可能低于同质嵌套

## 六、关键 bug 和解决方法

| 问题 | 症状 | 修复 |
|------|------|------|
| timedelta64 内存错误 | 子进程 `allocate timedelta64[D]` | compute_worker 跳过非浮点因子 |
| 缓存写入失败 | 内联路径组合无缓存 | `_add_to_history` 直接返回 ID，不查表达式 |
| 缓存读取越界 | 套娃返回 "no valid factors" | `f_train = cached_matrix` 不切片 |
| 分钟数据警告刷屏 | 日志/stdout 满 | 内联路径 stdout → devnull |
| Flask 未重启导致旧代码 | 修复后问题仍在 | `taskkill /F /IM python.exe` + 重启 |
| .pyc 缓存 | Flask 用旧字节码 | `find -name __pycache__ -delete` |
| 端口被占 | Flask `port already in use` | PowerShell `Get-NetTCPConnection` 找 PID 杀之 |

## 九、内存管理要点

1. **Flask 内联路径内存不归还 OS** — 每个组合计算后 Python 释放对象但不还内存，Flask RSS 只涨不跌
2. **定期重启 Flask** — 内存低于 1GB 或 Flask > 3.5GB 时杀掉重启
3. **僵尸 Python 进程** — 定期检查并清理（`psutil` + `taskkill`）
4. **不要同时跑两个 pipeline** — Flask + 独立脚本不能同时加载 pipeline，否则双份 ~4GB
5. **清除 .pyc 缓存** — 修改代码后删除 `__pycache__` 目录避免 Flask 用旧版本
